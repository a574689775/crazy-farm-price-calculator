## 技术方案：手机号登录（基于 Supabase + 腾讯云短信）

本文档描述在现有基于 Supabase Auth 的邮箱登录体系上，新增「中国大陆手机号登录」能力的整体技术方案。设计原则：

- **登录统一为「账号 + 密码」**：手机号登录 = 手机号 + 密码，与邮箱登录交互一致；**登录不发短信**，成本可控。
- **短信仅用于**：手机号注册、手机号找回密码、个人中心绑定手机号。
- **沿用 Supabase 作为统一用户系统**，使用腾讯云短信发送国内验证码。

---

## 一、整体架构概览

### 1.1 现状概述

- 用户体系：Supabase Auth（邮箱 + 密码），扩展信息在 `user_profiles` 等表。
- 登录方式：邮箱 + 密码；注册用邮箱验证码；找回密码用邮箱验证码。

### 1.2 新增能力概览

- **前端登录页**
  - 「手机号登录」Tab：输入**手机号 + 密码**，与邮箱登录并列（手机号推荐）。
  - 「手机号注册」：手机号 + 短信验证码 + 密码 + 确认密码 + 邀请码（选填），交互与邮箱注册一致。
  - 「手机号找回密码」：手机号 + 短信验证码 + 新密码 + 确认密码，交互与邮箱找回一致。
- **后端（Supabase Edge Functions）**
  - `send-sms-code`：发送短信验证码，`scene` 仅支持 `register`、`reset_password`、`bind`。
  - `verify-sms-code`：校验验证码；`register` 创建账号并返回 session，`reset_password` 重置密码，`bind` 绑定手机号到当前用户。
  - `get-email-by-phone`：根据手机号查询对应用户的虚拟邮箱，供前端「手机号+密码」登录时调用 `signInWithPassword`。
- **数据层**
  - `user_profiles` 中手机号字段及唯一约束（一号一用户）。
  - `sms_codes` 表存验证码，用于注册/找回/绑定流程。

---

## 二、数据表与账号映射

### 2.1 `user_profiles` 扩展

- `phone`：中国大陆 11 位手机号，`UNIQUE`。
- `phone_verified_at`：验证通过时间。
- 可选：`sms_login_enabled`。

### 2.2 `sms_codes` 表

- `phone`、`code`、`scene`（`register` | `reset_password` | `bind`）、`created_at`、`expires_at`、`used_at`、`failed_attempts`、`ip` 等。
- 单手机号 1 分钟 1 条；单 IP 10 分钟最多 3 条。

---

## 三、流程设计

### 3.1 手机号登录（账号 + 密码，不发短信）

1. 用户输入手机号、密码，点击「登录」。
2. 前端调用 `get-email-by-phone(phone)`，获取该手机号对应的虚拟邮箱（如 `phone_13800138000@crazyfarm.local`）。
3. 若返回「该手机号未注册」，提示用户先注册。
4. 若返回 `email`，前端调用 `supabase.auth.signInWithPassword({ email, password })` 完成登录。

### 3.2 手机号注册（发短信）

1. 用户输入手机号，点击「发送验证码」；前端调用 `send-sms-code`，`scene: 'register'`。
2. 用户输入验证码、密码、确认密码、邀请码（选填），点击「注册」。
3. 前端调用 `verify-sms-code`，`scene: 'register'`，body 含 `phone`、`code`、`password`。
4. 后端校验验证码，若该手机号未注册则：创建 `auth.users`（虚拟邮箱 + 用户设置的密码）、插入 `user_profiles`，生成 session 返回。
5. 前端 `setSession`，可选 `assignRandomDisplayName`、`bindInviteRelation(inviteCode)`，然后进入主应用。

### 3.3 手机号找回密码（发短信）

1. 用户在「手机号找回密码」页输入手机号，点击「发送验证码」；前端调用 `send-sms-code`，`scene: 'reset_password'`。
2. 用户输入验证码、新密码、确认密码，点击「重置密码」。
3. 前端调用 `verify-sms-code`，`scene: 'reset_password'`，body 含 `phone`、`code`、`newPassword`。
4. 后端校验验证码，根据手机号查 `user_profiles` 得到 `user_id`，调用 `auth.admin.updateUserById(userId, { password })`。
5. 前端可选：用 `getEmailByPhone(phone)` + `signIn(email, newPassword)` 自动登录，或仅提示「请使用新密码登录」。

### 3.4 老用户绑定手机号（个人中心，发短信）

1. 已登录用户在个人中心点击「绑定手机号」，输入手机号，点击「发送验证码」；前端调用 `send-sms-code`，`scene: 'bind'`。
2. 用户输入验证码，点击「确认绑定」；前端调用 `verify-sms-code`，`scene: 'bind'`，请求头带 `Authorization: Bearer <access_token>`。
3. 后端从 JWT 解析当前用户，校验验证码，确认该手机号未被其他用户占用后，更新当前用户的 `user_profiles.phone` 等。
4. 绑定成功后，该用户可使用「手机号+密码」登录（密码即其原有账号密码；若为纯邮箱账号则用邮箱密码）。

## 四、安全与风控

- **短信限流**：单手机号 1 分钟 1 条；单 IP 10 分钟 3 条。
- **验证码**：10 分钟有效；单条验证码错误 ≥5 次作废。
- **绑定场景**：当前用户 ID 仅从 JWT 获取，不信任前端传入。
- **get-email-by-phone**：仅返回已注册手机号对应的虚拟邮箱，未注册返回通用错误，避免枚举。

---

## 五、Edge Functions 一览

| 函数 | 作用 | scene / 说明 |
|------|------|----------------|
| send-sms-code | 发送短信验证码 | `register`、`reset_password`、`bind` |
| verify-sms-code | 校验验证码并执行业务 | `register`：创建账号并返回 session；`reset_password`：重置密码；`bind`：绑定手机号 |
| get-email-by-phone | 根据手机号查虚拟邮箱 | 无 scene，POST body：`{ phone }`，返回 `{ ok, email? }` |

---

## 六、前端改造要点

- 登录页：手机号 Tab 为「手机号+密码」表单；邮箱 Tab 保持原样。
- 注册：手机号注册与邮箱注册并列，手机号注册含「验证码、密码、确认密码、邀请码（选填）」；邮箱注册含「验证码、邀请码（选填）、密码、确认密码」。
- 找回密码：支持「邮箱找回」与「手机号找回」两种入口，表单与注册风格一致。
- 邀请码：邮箱注册、手机号注册均提供「邀请码（选填）」；注册成功后调用 `bindInviteRelation`。
- 本地记忆：成功登录后可将手机号/邮箱写入 localStorage，下次自动带出。

---

## 七、小结

- **登录**：手机号与邮箱均为「账号+密码」，不发短信，成本可控。
- **短信**：仅在注册、找回密码、绑定手机号时发送。
- **交互**：手机号与邮箱在注册/找回上的表单项与流程对齐，邀请码在两套注册中均可使用。
